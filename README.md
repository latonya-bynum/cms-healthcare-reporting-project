# CMS Healthcare Reporting Project

## Overview
This project demonstrates a healthcare reporting workflow using CMS Medicare Provider Utilization data. The goal is to analyze provider-level service volume and generate structured outputs.

## Objective
To identify high-utilization providers and simulate real-world CMS reporting use cases.

## Tools & Technologies
- Python (pandas)
- Jupyter Notebook
- SQL (conceptual workflow)
- CMS Medicare Data

## Key Tasks
- Loaded and explored CMS dataset
- Grouped data by National Provider Identifier (NPI)
- Calculated total services per provider
- Ranked providers by service volume
- Generated top provider report

## Output
- `top_providers.csv` → Ranked list of providers by total services

## Relevance
This project demonstrates:
- Data validation and review
- Healthcare reporting workflows
- CMS data familiarity
- Structured data output generation
